# Business Questions

The project answers **15 business questions**.

## Original Business Analysis

### Q1. Retrieve all sales made on 2022-11-05

### Objective

Retrieve every transaction completed on **5 November 2022**.

### SQL Query

```sql
SELECT *
FROM retail_sales
WHERE sale_date='2022-11-05';
```

### Explanation

This query filters the dataset and returns all transactions made on **5 November 2022**. It helps analyze daily sales performance.

---

## Q2. Find Clothing sales during November 2022 where quantity purchased is greater than or equal to 4.

### Objective

Identify high-volume Clothing sales during November.

### SQL Query

```sql
SELECT *
FROM retail_sales
WHERE category='Clothing'
AND TO_CHAR(sale_date,'YYYY-MM')='2022-11'
AND quantity>=4;
```

### Explanation

This query filters records for the Clothing category, checks whether the transaction occurred in November 2022, and returns only orders with a quantity of at least four.

---

## Q3. Calculate total sales for each product category.

### Objective

Compare revenue generated by different product categories.

### SQL Query

```sql
SELECT
category,
SUM(total_sale) AS net_sales,
COUNT(*) AS total_orders
FROM retail_sales
GROUP BY category
ORDER BY net_sales DESC;
```

### Explanation

The query groups transactions by product category and calculates total revenue and total orders for each category.

---

## Q4. Find the average age of customers purchasing Beauty products.

### Objective

Understand the average age of Beauty product customers.

### SQL Query

```sql
SELECT
ROUND(AVG(age),2) AS average_age
FROM retail_sales
WHERE category='Beauty';
```

### Explanation

The AVG() function calculates the average customer age for Beauty purchases while ROUND() limits the result to two decimal places.

## Q6. Count the total number of transactions made by each gender in every product category.

### Objective

Analyze purchasing behavior by comparing the number of transactions made by male and female customers across different product categories.

### SQL Query

```sql
SELECT
category,
gender,
COUNT(transaction_id) AS total_transactions
FROM retail_sales
GROUP BY category, gender
ORDER BY category, gender;
```

### Explanation

This query groups the dataset by **category** and **gender** and counts the number of transactions for each combination. The results help identify which gender contributes more transactions in each product category.

---

## Q7. Find the best-selling month of each year based on average sales.

### Objective

Identify the month that achieved the highest average sales during each year.

### SQL Query

```sql
SELECT
year,
month,
avg_sale
FROM
(
SELECT
EXTRACT(YEAR FROM sale_date) AS year,
EXTRACT(MONTH FROM sale_date) AS month,
AVG(total_sale) AS avg_sale,
RANK() OVER
(
PARTITION BY EXTRACT(YEAR FROM sale_date)
ORDER BY AVG(total_sale) DESC
) AS ranking
FROM retail_sales
GROUP BY
EXTRACT(YEAR FROM sale_date),
EXTRACT(MONTH FROM sale_date)
) t
WHERE ranking = 1;
```

### Explanation

This query uses the **RANK() Window Function** to rank months according to their average sales within each year. Only the month ranked first is displayed, making it easy to identify the best-performing month.

## Q10. Categorize orders into Morning, Afternoon, and Evening shifts.

### Objective

Analyze customer purchasing behavior based on the time of day.

### SQL Query

```sql
WITH hourly_sales AS
(
SELECT
*,
CASE
WHEN EXTRACT(HOUR FROM sale_time) < 12 THEN 'Morning'
WHEN EXTRACT(HOUR FROM sale_time) BETWEEN 12 AND 17 THEN 'Afternoon'
ELSE 'Evening'
END AS shift
FROM retail_sales
)

SELECT
shift,
COUNT(*) AS total_orders
FROM hourly_sales
GROUP BY shift
ORDER BY total_orders DESC;
```

### Explanation

A **Common Table Expression (CTE)** is used to classify every transaction into Morning, Afternoon, or Evening based on the sale time. The final query counts the number of orders in each shift, helping identify peak shopping periods.

## Q12. Find total revenue generated by each age group.

### Objective

Analyze revenue generated by different customer age groups.

### SQL Query

```sql
SELECT
CASE
    WHEN age BETWEEN 18 AND 25 THEN '18-25'
    WHEN age BETWEEN 26 AND 35 THEN '26-35'
    WHEN age BETWEEN 36 AND 45 THEN '36-45'
    ELSE '46+'
END AS age_group,
SUM(total_sale) AS total_revenue
FROM retail_sales
GROUP BY age_group
ORDER BY total_revenue DESC;
```

### Explanation

Customers are divided into different age groups using the **CASE** statement. The query calculates the total revenue generated by each group, helping identify the most valuable customer segment.

## Q13. Find the busiest shopping hour.

### Objective

Determine the hour of the day with the highest number of customer transactions.

### SQL Query

```sql
SELECT
EXTRACT(HOUR FROM sale_time) AS shopping_hour,
COUNT(*) AS total_orders
FROM retail_sales
GROUP BY shopping_hour
ORDER BY total_orders DESC;
```

### Explanation

The **EXTRACT()** function retrieves the hour from the sale time. Transactions are grouped by hour to identify peak shopping times.

## Q14. Find monthly revenue.

### Objective

Analyze revenue trends over different months.

### SQL Query

```sql
SELECT
DATE_TRUNC('month', sale_date) AS month,
SUM(total_sale) AS revenue
FROM retail_sales
GROUP BY month
ORDER BY month;
```

### Explanation

This query groups sales by month and calculates total monthly revenue. It helps identify seasonal trends and business growth over time.

Calculate the average amount spent per transaction.

### SQL Query

```sql
SELECT

```

### Explanation

The **AVG()** function calculates the average value of all transactions, while **ROUND()** formats the result to two decimal places. This metric is useful for evaluating customer spending behavior.
